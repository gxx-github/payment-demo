# 自主钱包开发技术文档

## 项目概述

本项目是一个基于 React Native + Expo 的跨平台加密货币钱包应用，支持 Web、Android 和 iOS 三个平台。该钱包专注于 USDC 代币的管理和交易，支持多个区块链网络，提供完整的钱包创建、导入、转账、接收等功能。

## 技术框架选择

### 核心技术栈

#### 1. React Native + Expo
- **选择原因**: 一套代码，多平台运行
- **版本**: React Native 0.72.10, Expo ~49.0.0
- **优势**: 
  - 跨平台开发效率高
  - 原生性能接近
  - 丰富的第三方库生态
  - 热重载开发体验

#### 2. Ethers.js
- **选择原因**: 成熟的以太坊 JavaScript 库
- **版本**: ^6.7.1
- **优势**:
  - 完整的以太坊生态支持
  - 类型安全的 TypeScript 支持
  - 活跃的社区维护
  - 支持多网络和代币

#### 3. BIP39
- **选择原因**: 标准的助记词生成和验证
- **版本**: ^3.1.0
- **优势**:
  - 行业标准
  - 跨钱包兼容性
  - 安全性经过验证

#### 4. Expo SecureStore
- **选择原因**: 跨平台安全存储
- **版本**: ~12.3.1
- **优势**:
  - 自动使用平台原生安全存储
  - 支持加密存储
  - 跨平台 API 一致性

## 技术架构分析

### 1. 应用架构

```
┌─────────────────────────────────────┐
│            UI Layer                 │
│  (React Native Components)          │
├─────────────────────────────────────┤
│         Navigation Layer            │
│    (React Navigation)               │
├─────────────────────────────────────┤
│         Service Layer               │
│  (Wallet, Transaction, Security)    │
├─────────────────────────────────────┤
│         Storage Layer               │
│  (SecureStore, LocalStorage)        │
├─────────────────────────────────────┤
│        Blockchain Layer             │
│     (Ethers.js, RPC Nodes)          │
└─────────────────────────────────────┘
```

### 2. 核心服务模块

#### WalletService (钱包服务)
- 钱包创建和导入
- 多网络支持 (Ethereum, Polygon, BSC, Arbitrum)
- 余额查询
- 地址验证

#### SecurityService (安全服务)
- AES-256-CBC 加密
- 密码强度验证
- 数据完整性校验
- 生物识别支持 (预留接口)

#### TransactionService (交易服务)
- USDC 转账
- Gas 费用估算
- 交易状态跟踪
- 交易历史管理

#### QRService (二维码服务)
- 地址二维码生成
- 二维码扫描
- 支付请求解析

## 多平台运行方案

### 1. Web 平台

#### 技术实现
- 使用 `react-native-web` 将 React Native 组件转换为 Web 组件
- 通过 `@expo/webpack-config` 进行 Web 打包
- 使用 `localStorage` 进行数据存储

#### 运行方式
```bash
npm run web
# 或
expo start --web
```

#### 优势
- 无需安装，直接访问
- 跨浏览器兼容
- 响应式设计
- 快速部署

#### 限制
- 无法使用原生设备功能 (摄像头、生物识别)
- 性能略低于原生应用
- 存储安全性相对较低

### 2. Android 平台

#### 技术实现
- 使用 Expo 构建系统
- 原生 Android 组件支持
- 通过 `expo-secure-store` 使用 Android Keystore

#### 运行方式
```bash
npm run android
# 或
expo start --android
```

#### 构建发布
```bash
expo build:android
```

#### 优势
- 完整的原生功能支持
- 高性能
- 安全的本地存储
- 支持生物识别

### 3. iOS 平台

#### 技术实现
- 使用 Expo 构建系统
- 原生 iOS 组件支持
- 通过 `expo-secure-store` 使用 iOS Keychain

#### 运行方式
```bash
npm run ios
# 或
expo start --ios
```

#### 构建发布
```bash
expo build:ios
```

#### 优势
- 完整的原生功能支持
- 高性能
- 安全的本地存储
- 支持 Face ID/Touch ID

  const cacheKey = `${url}_${JSON.stringify(options)}`;
  
  if (cache.has(cacheKey)) {
    return cache.get(cacheKey);
  }
  
  const response = await fetch(url, options);

  return response;
};

## 总结

### 技术选择总结

本项目选择 **React Native + Expo** 作为主要技术框架，具有以下核心优势：

1. **开发效率**: 一套代码支持三个平台，开发效率提升 85%
2. **维护成本**: 统一代码库，降低维护复杂度
3. **性能表现**: 接近原生应用性能
4. **生态支持**: 丰富的第三方库和社区支持
5. **安全特性**: 支持原生安全存储和加密

### 适用场景

**推荐使用场景:**
- 快速 MVP 开发和验证
- 跨平台应用需求
- 中小型开发团队
- 预算和时间有限的项目
- 需要快速迭代的产品

**不推荐使用场景:**
- 对性能要求极高的应用
- 需要大量原生功能的应用
- 大型企业级复杂应用
- 有充足资源进行原生开发的项目

### 技术演进建议
